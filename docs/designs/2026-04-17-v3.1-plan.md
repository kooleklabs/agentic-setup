## Chunk 3: `--execute` mode

### Task 5: `lib/gh-impl-branch.js` — branch + commit helpers

**Files:**
- Create: `lib/gh-impl-branch.js`
- Create: `lib/__tests__/gh-impl-branch.test.js`

Two exports:
- `createImplBranch({ cwd, slug, base })` — pre-flight checks (branch doesn't exist locally, doesn't exist remotely via `git ls-remote`, working tree clean) + `git checkout <base>` + `git checkout -b impl/<slug>`. Throws on any pre-flight failure.
- `commitImplChanges({ cwd, slug, issueNumber, featureName })` → `{ sha }` — `git add -A` then `git commit -m "wip(impl): <featureName> (refs #<N>)"` if there are staged changes. Returns the commit SHA. Returns `null` if nothing to commit (agent made no file changes).

- [ ] **Step 1: Write failing tests (~6 tests)**
  - `createImplBranch` happy path: 4 git calls (status, show-ref, ls-remote, checkout -b)
  - `createImplBranch` dirty working tree → throws
  - `createImplBranch` local branch exists → throws
  - `createImplBranch` remote branch exists → throws
  - `commitImplChanges` happy (staged changes) → returns SHA from `git rev-parse HEAD`
  - `commitImplChanges` no changes → returns `{sha: null}`

- [ ] **Step 2–5:** run failing → implement → green + lint → commit `feat: add gh-impl-branch helpers for v3.1 execute`.

### Task 6: `lib/impl-generator.js` — SDK call with tools enabled

**Files:**
- Create: `lib/impl-generator.js`
- Create: `lib/__tests__/impl-generator.test.js`

Exports:
- `runImplementation({ planBody, issue, architectureSection, apiSpecYaml, cwd, model, maxCostUsd, abort })` → `{ ok, body, filesWritten, commandsRun, costUsd } | { cancelled } | { ok: false, error }`.

Key implementation notes:
- **Tools enabled** — `tools: { type: 'preset', preset: 'claude_code' }`, `systemPrompt: { type: 'preset', preset: 'claude_code' }` (exactly opposite stance to `plan-generator`).
- Stream handler tracks tool uses by type: `{type: 'tool_use', name: 'Write', input: {file_path}}` → push to `filesWritten`; same for `Edit`; `Bash` → push command to `commandsRun`.
- Cost tracking: sum `message.usage` per assistant message, multiply by model's per-token rates (hardcode for Sonnet/Opus as of 2026; pull from env if present). Hard abort via `abort.abort()` when `costUsd > maxCostUsd`.
- `-cost-usd` estimate shown before invocation — rough input-token × rate.

Prompt (user) has exactly the structure from the design spec's §Implementation Prompt Structure.

- [ ] **Step 1: Write failing tests (~5 tests)**
  - Happy path: mock SDK stream yields `tool_use` + `assistant` text + `result` → returns `ok: true, filesWritten: [...], commandsRun: [...]`
  - SDK error → `{ok: false, error}`
  - Abort (simulated via `abort.signal.aborted`) → `{cancelled: true}`
  - Cost cap exceeded → abort triggered, `{cancelled: true}`
  - Prompt includes plan body verbatim

Mock `@anthropic-ai/claude-agent-sdk` with `jest.mock(...)` and `fakeStream(messages)` helper (reuse pattern from `plan-generator.test.js`).

- [ ] **Step 2–5:** run failing → implement → green + lint → commit `feat: add impl-generator SDK wrapper for v3.1 execute`.

### Task 7: Wire `--execute` flag + route through `lib/github-sync.js`

**Files:**
- Modify: `lib/github-sync-cli.js`
- Modify: `lib/github-sync.js`
- Modify: `lib/__tests__/github-sync-cli.test.js`
- Modify: `lib/__tests__/github-sync.test.js`

CLI changes:
- Add `--execute`, `--impl-branch <name>`, `--max-cost-usd <n>` to `parseArgs`
- Default `maxCostUsd: 5.00`; default `implBranch: undefined` (orchestrator picks `impl/<slug>`)

Orchestrator changes — `runGithubSync` routes on `opts.execute`:
- `false` (default) → existing plan flow
- `true` → new `runExecute(opts)` path:
  1. Reuse fetchFeatureIssue + parseArchitecture validations
  2. Call `readMergedPlan` — fail early if missing or validator fails
  3. Verify plan PR is merged: check `plan/<slug>` branch is NOT on origin (via `git ls-remote`). Fail if still there.
  4. Pre-flight via `createImplBranch` (creates the branch if checks pass)
  5. Print plan summary + cost estimate
  6. `--dry-run` short-circuit — print prompt + estimate, no SDK call
  7. Approval (or `--yes`)
  8. `runImplementation(...)` — SDK call
  9. On cancel / error: branch is left in place with any files the agent wrote — user inspects
  10. On success: `commitImplChanges(...)` produces the WIP commit
  11. Print banner with next steps (git diff / git push / gh pr create)

- [ ] **Step 1: Write failing tests**

CLI (~5 new tests in `github-sync-cli.test.js`):
- `--execute` sets `opts.execute = true`
- `--impl-branch feature-foo` captures the name
- `--max-cost-usd 10` parses to `10` (Number)
- `--max-cost-usd 0` throws (must be positive)
- `--execute --dry-run` combined works

Orchestrator (~8 new tests in `github-sync.test.js`):
- `--execute` routes to execute branch
- Pre-flight missing plan file → `{ok: false, error}` with guidance
- Pre-flight plan PR still open → `{ok: false, error}`
- Pre-flight impl branch exists → `{ok: false, error}`
- `--execute --dry-run` — no SDK call, no branch created
- Happy path: calls runImplementation, then commitImplChanges, returns `{ok, plan.branch, plan.sha}`
- SDK error during runImplementation — no commit, branch left in place
- `--yes` bypasses the approval prompt in execute mode (already tested in plan mode)

- [ ] **Step 2: Run — expect failure**

```bash

```

- [ ] **Step 3: Implement**

Key structural choice: `runExecute` is a private helper function inside `github-sync.js`, not a separate exported module. `runGithubSync` stays the single public entry point; the execute branch is ~80 lines that reuse most of the plan-mode helpers.

- [ ] **Step 4: Full tests + lint**

## Appendix: Task ordering rationale

Why Chunk 2 (ergonomics) ships before Chunk 3 (execute)?

- `--yes` is needed by the orchestrator's execute tests anyway (we mock approval with `autoApprove`, but the CLI path should be tested end-to-end too)
- `detectDefaultBranch` is needed by Chunk 3's pre-flight (create impl branch FROM default branch)
- Both ergonomics are trivial; shipping them first reduces merge-conflict surface when Chunk 3's bigger edits land

---

## Appendix: Why no real-SDK integration test in CI

`impl-generator` calls Claude with real tools. Automated tests mock the SDK. Real-SDK E2E is gated to Step 3 of the release checklist (manual, with live credentials). Mirrors v2.6's architect-gate and v3.0's plan-generator posture: unit-test the message parsing + cost accounting + abort logic, E2E the behaviour pre-release.

Cost per real-SDK E2E run of `--execute`: estimated $0.50–$3.00 depending on feature complexity. Stays within `--max-cost-usd 5.00` default.

## Appendix: Subprocess safety (same as v2.7 / v3.0)

Every `gh` and `git` invocation goes through `spawnSync('<cmd>', [argsArray], { encoding: 'utf8' })` — no shell, no string interpolation. This is enforced by reusing `runGh` from `lib/github-issues.js` and the `runCmd` pattern from `lib/gh-plan-pr.js` for the new `gh-impl-branch.js` and `default-branch.js` modules.
